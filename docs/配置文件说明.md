# 配置文件说明

`redculture_server` 启动时会优先加载 YAML 配置文件，默认查找 `config/app.yaml`。这样数据库连接、HTTP 端口、日志输出、JWT、AI/TTS 等运行参数可以和可执行文件解耦。

## 加载顺序

优先级从低到高：

| 优先级 | 来源 | 说明 |
| --- | --- | --- |
| 1 | `config/app.yaml` | 常规配置来源 |
| 2 | 环境变量 | 适合生产环境覆盖密钥、数据库连接 |
| 3 | 命令行参数 | 适合临时调试覆盖 |

## 配置文件位置

默认会依次尝试：

| 顺序 | 路径 |
| --- | --- |
| 1 | 当前工作目录下的 `config/app.yaml` |
| 2 | 可执行文件旁边的 `config/app.yaml` |
| 3 | CLion 常见构建目录回溯到项目根目录的 `config/app.yaml` |

也可以显式指定：

```bash
./redculture_server --config /home/chen/Code/service/config/app.yaml
```

## 数据库配置

本地开发示例：

```yaml
storage:
  enabled: true
  postgres_uri_env: "RCS_POSTGRES_URI"
  postgres_uri: "postgresql://postgres:123456@127.0.0.1:5432/redculture"
  auto_migrate: true
  allow_without_storage: true
```

说明：

| 字段 | 作用 |
| --- | --- |
| `enabled` | 是否启用 PostgreSQL 存储 |
| `postgres_uri_env` | 如果该环境变量存在，会覆盖 `postgres_uri` |
| `postgres_uri` | 默认 PostgreSQL 连接串 |
| `auto_migrate` | 连接成功后是否自动建表/迁移 |
| `allow_without_storage` | 数据库异常时，部分互动流程是否继续返回成功 |

注册和账号密码登录强依赖数据库。如果数据库连接失败，注册接口会返回 503。

## 日志配置

```yaml
logging:
  service_name: "red_culture_service"
  logger_name: "redculture"
  level: "info"
  console: true
  json: false
  file_enabled: true
  file_path: "logs/redculture_server.log"
  pattern: "[%Y-%m-%d %H:%M:%S.%e] [%n] [%^%l%$] %v"
```

说明：

| 字段 | 作用 |
| --- | --- |
| `level` | 日志级别，支持 `trace/debug/info/warn/error/critical` |
| `console` | 是否输出到控制台 |
| `json` | 是否输出 JSON 日志内容 |
| `file_enabled` | 是否写入日志文件 |
| `file_path` | 日志文件路径，支持相对路径 |
| `pattern` | spdlog 日志格式 |

当前 HTTP 访问日志、业务错误日志、启动日志都会走同一个 spdlog 默认 logger，因此会同时遵守 YAML 中的控制台/文件/级别配置。

## HTTP 服务配置

```yaml
server:
  listen_address: "0.0.0.0"
  port: 8080
  thread_count: 2
  max_body_bytes: 1048576
  enable_cors: true
```

`network` 段仍保留给 TCP 长连接接入层；Unity 当前 HTTP 联调主要使用 `server` 段。

## 临时覆盖示例

使用配置文件启动：

```bash
/home/chen/Code/service/build/debug/app/redculture_server/redculture_server \
  --config /home/chen/Code/service/config/app.yaml
```

临时覆盖端口：

```bash
/home/chen/Code/service/build/debug/app/redculture_server/redculture_server \
  --config /home/chen/Code/service/config/app.yaml \
  --port 8081
```

生产环境覆盖敏感信息：

```bash
export RCS_POSTGRES_URI='postgresql://postgres:真实密码@127.0.0.1:5432/redculture'
export RCS_JWT_SECRET='更长的生产密钥'
/home/chen/Code/service/build/debug/app/redculture_server/redculture_server
```
